# AI Codebase Understanding Assistant

**Codebase:** Sleep Journal Sample Project (provided)  
**Timebox:** 4-5 hours total  
**Submission:** Git repo or zipped folder + README

---

## What You'll Build

You'll build a small **AI-assisted developer tool** that helps an engineer quickly understand an existing mobile codebase.

The provided Sleep Journal project is intentionally imperfect. It contains bugs, inconsistencies, and mixed patterns similar to what engineers encounter in real-world projects.

Your tool should analyze this codebase and produce **structured insights** that help a developer:

- Understand the high-level architecture
- Identify likely issues or risks
- Know what questions to ask before making changes

**You are not expected to fix the code.** The goal is to accelerate understanding, not refactor the app.

---

## Expectations

### Core Requirements

Your submission must:

- Be runnable (CLI, script, or agentic skill)
- Analyze **multiple files** in the codebase (not single-file only)
- Produce a **machine-readable JSON report**
- Use AI thoughtfully (LLMs, agents, embeddings, etc. are allowed)
- Clearly explain decisions, limitations, and tradeoffs

You may use any programming language, framework, or AI tooling (Copilot/OpenAI preferred).

---

## Output

Your tool should generate a JSON report that includes:

- A short high-level summary of the codebase
- A list of findings, each with:
  - Type (bug, smell, architecture, test gap, documentation gap, uncertainty)
  - Location (file + line range or component)
  - Explanation
  - Confidence level (high / medium / low)
- A short list of questions a new engineer should ask
- Known limitations of your approach

The exact format is flexible as long as the output is structured and readable.

---

## What to Submit

Please include:

1. **Code**
   - Runnable entry point
   - Dependency file (`requirements.txt`, `package.json`, etc.)
2. **README** (required)
   - What problem you chose to focus on
   - How your tool works (high-level)
   - How to run it locally
   - AI tools/models used (if any)
   - Tradeoffs and known limitations
3. **Sample Output**
   - A JSON report generated by running your tool against the Sleep Journal project

---

## Guidance

- We care more about how you think than how much you build, so the project is designed to take no more than 4 to 5 hours of your time.
- You are encouraged to use AI. Be transparent about how you used it.
- Be honest about what you didn't finish and why.

---

## How We'll Evaluate

We'll look at:

- Clarity of problem framing and scope
- Quality of engineering decisions
- Thoughtful and responsible use of AI
- Ability to explain tradeoffs and limitations
- Code readability and reproducibility

Feature completeness matters less than clear reasoning.
